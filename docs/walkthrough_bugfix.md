# 기능 보완 및 버그 픽스 보고서

요청하신 4가지 수정 사항을 모두 완료하였습니다! 아래의 변경 내용 및 검증 결과를 확인해 주세요.

## 1. 주식 목록 데이터 출력 오류 해결
- **문제 원인**: 서버에서 `transactions` 테이블을 찾지 못해 500 오류가 발생하면서 주식 데이터를 불러오지 못하고 있었습니다. (실제 데이터베이스 테이블은 `stock_transactions`임)
- **해결 방법**: 
  - 엔티티의 이름 매핑을 `@Table(name = "stock_transactions")`으로 정상화했습니다.
  - MyBatis의 `stockDetailMapper.xml` 및 `myPointHistoryMapper.xml`의 모든 `transactions` 참조를 `stock_transactions`로 수정하였습니다.

## 2. 뉴스와 쿠폰 상점 리스트 중복 출력 해결
- **문제 원인**: Spring Boot 실행 시마다 `DataInitializer`가 중복해서 데이터를 삽입하는 문제가 있었습니다.
- **해결 방법**:
  - `DataInitializer`에 테이블 내부의 개수를 파악하고, `COUNT`가 0일 때만 초기 데이터를 추가하도록 로직을 수정했습니다.
  - 현재 저장된 기존의 중복 데이터(아이디가 3, 4, 5번 이후인 데이터들)를 모두 정리하였습니다.

## 3. 포인트 내역 미출력 현상 수정
- **문제 원인**: React에서 포인트 내역 데이터 자체가 없을 때 (`null` 반환 시) `length` 속성에 접근하려다가 프론트엔드가 크래시(하얀 화면)되는 문제가 있었습니다.
- **해결 방법**:
  - `PointsHistory.jsx` 에서 데이터를 매핑할 때, 옵셔널 체이닝 및 null 체크(`history && history.length > 0`)를 추가하여, 데이터가 없는 경우 안전하게 "포인트 내역이 없습니다." 라는 메시지를 띄우게 처리했습니다.

## 4. 내 자산 페이지 변경 및 보유 쿠폰 기능 추가
- **문구 수정**: 대시보드의 자산 카드 라벨을 요청하신 대로 [총 자산, 보유 포인트, 총 손익, 보유 쿠폰]으로 변경했습니다.
- **쿠폰 카드 클릭 활성화**:
  - `Dashboard.jsx`의 "보유 쿠폰" 카드를 클릭 시 `/my-coupons`로 이동하게 적용하였습니다.
- **보유 쿠폰 전용 페이지 (`MyCoupons.jsx`) 신설**:
  - 사용자가 구매한 쿠폰 리스트와 상태(`사용완료`/`사용가능`)를 조회할 수 있는 전용 UI를 구현하고, 다크모드 및 프리미엄 글래스모피즘 컨셉의 `MyCoupons.css`를 연결하였습니다.
- **백엔드 API 추가**:
  - `CouponController`에 `/api/coupons/my` 엔드포인트를 추가하였습니다.
  - `couponMapper.xml`을 업데이트하여 쿠폰의 이름, 가격, 날짜뿐만 아니라 `state` (상태)까지 프론트엔드로 전달하게끔 쿼리를 보완했습니다.

> [!TIP]
> 이제 백엔드 서버 구동(`npm run dev`, `spring-boot:run`)을 통해 브라우저에서 로그인(`abc`, `abc123!`) 하신 후 대시보드의 **보유 쿠폰** 카드를 클릭해 보세요! 변경된 사항들을 매끄럽게 테스트하실 수 있습니다.
