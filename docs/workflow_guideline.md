# 프로젝트 작업 진행 가이드라인 (Workflow Guideline)

앞으로의 모든 작업은 반드시 아래의 6단계 프로세스를 엄격하게 준수하여 진행합니다.

1. **계획 세우기 (Plan)**
   - 작업을 시작하기 전, 무엇을 어떻게 수정할지 분석하고 구체적인 계획서(예: `implementation_plan.md`)를 작성합니다.
   - 사용자에게 계획에 대한 승인을 받습니다.

2. **체크리스트 작성하기 (Checklist)**
   - 승인된 계획을 바탕으로 해야 할 작업 단위를 잘게 쪼개어 체크리스트(예: `task.md`)를 작성합니다.

3. **작업 진행 (Execute)**
   - 체크리스트에 따라 실제 코드 수정 및 파일 조작 작업을 진행합니다.
   - 작업 중 예상치 못한 이슈가 발생하면 체크리스트에 추가하고 대응합니다.

4. **제대로 작업했는지 2차 확인 (Verify)**
   - 작업이 끝난 후, 코드가 정상적으로 빌드 및 실행되는지, 요구사항이 정확히 반영되었는지 테스트 및 검증합니다.

5. **작업 내용 정리 후 md 파일 제작 (Document)**
   - 완료된 작업 내역을 정리하여 `docs` 폴더 내에 개별 문서(예: `작업명.md` 또는 `walkthrough.md`)로 저장합니다.

6. **프로젝트 전체 과정 기록 파일 갱신 (Update Progress)**
   - 개별 작업이 끝나면 전체 진행 상황을 담고 있는 `PROGRESS.md` 및 `README.md` 등의 마스터 문서를 최신화합니다.
